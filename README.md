# 巡回ログ（スマホ対応版）

展示会の巡回ログアプリを **スマホから使える** ようにしたものです。
`index.html` 単体でも動きますが、GitHub Pages で公開すると次の利点があります。

- スマホのブラウザで URL を開くだけで使える（ファイル転送不要）
- 記録がその端末に自動保存される（`localStorage`。緑の「保存OK」バッジが出ます）
- **ホーム画面に追加すると、アプリのように全画面で起動**
- **一度開けばオフラインでも起動**（会場の電波が弱くても記録できる／PWA）

---

## セットアップ（初回だけ・1〜2分）

GitHub Pages を有効化すると、公開URLが発行されます。

### 手順
1. このリポジトリを `main`（または `master`）ブランチに反映する
   - このブランチ（`claude/mobile-compatibility-0sr8ek`）から Pull Request を作ってマージすればOK
2. GitHub の **Settings → Pages** を開く
3. **Build and deployment → Source** を **「GitHub Actions」** に設定
4. 数十秒〜1分ほどで公開されます。URLは次の形になります：

   ```
   https://riecle.github.io/Tenjikai/
   ```

   （正確なURLは Settings → Pages の上部、または Actions の "Deploy to GitHub Pages" のログに表示されます）

> すぐ試したい場合：Actions タブ → 「Deploy to GitHub Pages」→ **Run workflow** で、
> このブランチを選んで手動実行することもできます。

---

## スマホでの使い方

1. スマホのブラウザ（iPhone は **Safari**、Android は **Chrome**）で上記URLを開く
2. 画面上部に緑色の **「保存OK — この端末に自動保存中」** が出ていれば、記録は自動保存されます
3. **ホーム画面に追加**しておくと、次回からアプリのように開けます
   - **iPhone（Safari）**：共有ボタン `⬆︎` → 「ホーム画面に追加」
   - **Android（Chrome）**：右上メニュー `⋮` → 「ホーム画面に追加」／「アプリをインストール」
4. 一度開いておけば、**オフラインでも起動**します（会場で電波が弱くても安心）

### データについて
- 記録は **開いた端末のブラウザ内** に保存されます（サーバーには送られません）
- 別の端末とは共有されません。バックアップは画面右上の **「書き出し」→ JSONL／まとめコピー** を使ってください
- iPhone の **プライベートブラウズ**では自動保存が効かない場合があります（赤いバッジが出たら通常モードで開き直してください）

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | アプリ本体 |
| `manifest.webmanifest` | PWA設定（ホーム画面追加・全画面表示） |
| `sw.js` | サービスワーカー（オフライン起動用のキャッシュ） |
| `icons/` | ホーム画面用アイコン |
| `.github/workflows/pages.yml` | GitHub Pages への自動デプロイ |
| `.nojekyll` | Pages が余計な変換をしないための空ファイル |

## 更新方法
`index.html` を変更して `main` に反映すると、自動で再デプロイされます。
スマホ側は再読み込みすると新しい版に更新されます。
