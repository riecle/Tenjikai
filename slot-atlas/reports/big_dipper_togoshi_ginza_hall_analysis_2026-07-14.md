# BIGディッパー戸越銀座店 本解剖 — 2026-07-14

## 結論

この店は13店目として**予測解禁**する。ただし「告知が多いから毎日強い」とは扱わない。

- 通常日: **NO BET**。保守的通常平均+50.9枚は非等価の判定床+50をほぼ越えず、尾山台からの移動ペナルティ8点を引くと効用-7.1。
- 1日・20日: **A**。縮約後+169.6枚、移動後効用+111.6。ただし祝日・連休は降格し、2026-07-20（海の日）はB。
- 3・5・7・9のつく日: **B**。通常日との差は小さいため「相対強日」ではなく、絶対水準が判定床を越える日として限定採用。
- 11日・22日、月=日: **NO BET**。公開上の強日候補を実測で反転した回避日。
- 12日・16日: 数字は強いが事前登録のない探索後発見。媒体名が同じ条件で追試されるまで固定日へ昇格しない。
- 固定末尾: **棄却**。日付末尾と台番末尾の一致は正符号51.1%、非一致52.6%で優位なし。
- 機種: 前日公開優秀機種の翌日継続は16.4%、前日非選抜は6.4%。これは店ランクではなく、当日の機種優先順位にだけ使う。

## 取得データと負日復元

[みんレポ一覧](https://min-repo.com/tag/%e3%83%93%e3%83%83%e3%82%af%e3%83%87%e3%82%a3%e3%83%83%e3%83%91%e3%83%bc%e6%88%b8%e8%b6%8a%e9%8a%80%e5%ba%a7%e5%ba%97/)は負の店総差枚を `-` と表示する。このまま集計すると正の日だけが残り、平均が上振れる。

そこで直近225日（2025-12-01〜2026-07-13）について詳細ページを取得し、次のように分離した。

- 正の日146日: 一覧の公開総差枚・平均差枚をそのまま保存。
- 負の日79日: 機種別の平均Gと出率から `平均G × 3 × (出率/100 − 1)` を計算し、設置台数で加重して店平均を復元。
- 検算: 同じ式を公開正日へ適用した平均誤差+0.018枚、MAE 0.435枚。
- 機種日次20,229行、末尾日次2,475行も別テーブルへ保存。
- 一覧には400リンクあったが、負値を同品質で復元できない過去175日はモデルへ混ぜなかった。

復元式、ハッシュ、行数は `big_dipper_collection_summary_2026-07-14.json` と `source_snapshots` に保存した。

## 媒体告知を固定日に混ぜない

Hall Naviの公開過去スケジュールでは、2026-06-11〜07-13の33日中31日に何らかの告知がある。例えば7/1はフェノメノン等、7/7は複数のAI予想と来店取材、7/12はPlus Ultra取材等が重なっていた。

そのため、次を固定した。

1. 媒体名は `calendar_flags` へ事前登録する。
2. 媒体の全国平均やHall Navi評価を店平均へ加算しない。
3. 同一媒体・単独開催・同一貸玉・同じ事前公約でnを作る。
4. 12日・16日の高平均は、同じ媒体条件が付いた時だけ追試する。

2026-07-14〜18の公開予定はフラグとして保存済み。特に7/16は固定日としてはNO BETのまま、媒体条件の追試日として扱う。

## 位置レイヤー

負の末尾平均は公開ページで数値が伏せられるため、差枚の推測補完はしていない。正なら数値表示、負ならNULLとして符号だけを使った。

- 日付末尾一致: 46/90=51.1%
- 同じ日の非一致末尾: 426/810=52.6%
- 差: -1.48ポイント

よって「7の日は末尾7」のような固定末尾は使わない。島端・角・通路側も全台の位置履歴を同じ品質で保存できていないため未解禁。実戦では機種まで絞り、位置は当日の確定示唆か全台データ取得後に判断する。

## 実戦プロトコル

1. 日付ランクを確認する。通常、11/22、月=日は行かない。
2. 当日朝、公式告知とHall Naviの媒体名が事前登録どおりか確認する。媒体名だけでランクを上げない。
3. 前日公開優秀機種が現行設置に残っていれば、第一候補群より先に1段上げる。
4. 多台数の第一候補を優先し、1台機種の高平均を本命にしない。
5. 固定末尾、角、ゾロ目へ後付けで寄せない。
6. 抽選負けまたは第一候補が取れない時は、B日は無理に代替台へ座らず撤退。Aの1日・20日でも2巡目までに根拠台が空かなければ他市場へ切り替える。
7. 翌日、店平均・平均G・台勝率・媒体実施・優秀機種・末尾符号を `validation_log` へ入れ、confirm / kill / reframeを更新する。
