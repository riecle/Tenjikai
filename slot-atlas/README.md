## 365日を「必ず店名」で埋めない理由

統合版v0.9.8では、アップロードされた別系統v0.9.5の蒲田データを保持したままv0.9.7へ統合しました。マルハン蒲田1は、日次776行を持つ実測済み・予測解禁版を優先し、11日・30日・7の日など7本のルールを復元。楽園蒲田店は日次193行と4本の旧レジームルールを保存しつつ、2026/7/7の567→587台増台後はサンプル不足のため予測未解禁を維持しています。登録店舗は33店です。

全日へ強制的に店名を割り当てると、全店マイナスの日にも偽の狙い目を作ります。このDBでは毎日を必ず埋めますが、値は次のどちらかです。

- `S/A/B`: 狙い目として採用
- `C/NO BET`: 相対首位の店名を保持しつつ、結論は**見送り**

「行かない」を予測クラスに含めることで、365日カレンダーが選別器になります。

## データの層

1. `source_snapshots`: 取得時刻、URL、ハッシュ、失敗も保存
2. `hall_days`: 店×日
3. `machine_days`: 店×日×機種
4. `tail_days`: 店×日×末尾
5. `machine_scores`: 期間A、特定日選抜B、モメンタムC、統合S
6. `position_signals`: 公開レポートの機種×台番号ブロック。貸玉区分不明も明示
7. `calendar_flags`: 祝日、連休、周年、来店・取材
8. `evidence_rules`: n付きの現行仮説。棄却ルールも保存
9. `predictions`: **実測より先に固定した予測**
10. `validation_log`: kill / confirm / reframe

生データと派生指標を分離し、過去予測を上書きしません。これが「後から当たっていた指標だけ拾う」リークを防ぎます。

## スコア（v0.9で継続）

特定日の平均を店の通常ベースへ `prior_n=4` で縮約し、非等価店には運用上のプラス余白を引きます。

```text
予測平均 = (日タイプ平均 × n + 通常平均 × 4) / (n + 4)
調整エッジ = 予測平均 − 店舗別decision_floor
```

サンプル数、データ鮮度、日曜・祝日・連休を信頼度とランクへ反映します。n=1周年が即Sにならず、データ収集が止まった店も自動降格します。

さらに、基準日から90日を超える予測は1段階、180日超は2段階落とします。365日表の遠い将来は「現在の周期を置いたテンプレ」であり、毎日の再計算によって近づいた日だけが実戦ランクへ昇格します。

レッドロックで新たに「旧イベ×曜日」の交互作用を実装しました。6/9系を一括採用せず、26日を分離し、土曜と重なる場合だけ上位ルールが優先されます。公開機種月間表の鮮度が不整合だったため、Aは欠損のまま保存し、B/Cだけの暫定スコアを別注記しています。

ヒノマル中目黒では `forecast_enabled=false` を導入しました。営業中の店を閉店扱いせずDBに残しながら、日次n=0の段階では365日すべてを強制的に `NO BET` にできます。P'sCubeの毎日観測を蓄積し、通常日n≥30かつ主要日タイプn≥6を満たした時点で初回検定を行います。

オリパサ自由が丘も `forecast_enabled=false` ですが理由は異なります。日タイプ集計は存在する一方、20円/5円の率区分と通常日母集団が欠けています。`position_signals` は保存しても、店単位の365日予測には混ぜません。周年もDMMの9/14とHall Naviの9/15が不一致なので両日を `calendar_flags` へ候補登録し、現行実測が付くまでpendingです。

新宿東宝は通常ベース自体が高いため、7のつく日の絶対平均+98.1（n=35）を採用しつつ「曜日調整後の相対リフト」は棄却しています。31日は30日と分離して+337.4（n=7、7戦全勝）。末尾6・7・下2桁ゾロ目の強さは `position_signals` へ保存しますが、店全体が強い日の保証とは扱いません。RAW ExcelはCloudflare 403を取得失敗として記録し、公開HTMLで確認できない値を補完していません。

## 毎日の更新契約

毎朝の処理は次の順序で固定します。

1. 前日分をソース別に取得し、取得失敗も `source_snapshots` へ記録
2. 同じ店・日でもソースを混ぜず `hall_days` へ追記
3. 台数変更・入替・収集停止を検知
4. 直近レジームで日タイプ・機種・末尾を再計算
5. 当日以降365日の予測を新しい `model_run` として追加
6. 前日に固定した予測を実測と照合し、validation_logを更新
7. 当日のS/A/Bだけ通知。全店C以下なら「今日は見送り」

CSV取り込み列の最小契約:

```text
hall_id,result_date,avg_diff,avg_games,source_name
```

任意列は `total_diff,machine_win_rate,winning_units,total_units`。取り込み例:

```bash
python3 slot_atlas.py import-hall-days daily.csv
```
